Javascript functions as arguments

const differentiate = (fun, x) => { const h = 0.0001 return x => (fun(x + h) - fun(x)) / h } const f = x => 2 * x const fPrime = differentiate(f) // Since f(x) = 2x, we should expect f' = 2. console.log("f'(1) = " + fPrime(1)) // 2 console.log("f'(2) = " + fPrime(2)) // 2 const g = x => x * x const gPrime = differentiate(g) // Since g(x) = x^2, we should expect g' = 2x console.log("g'(1) = " + gPrime(1)) // 2*1 = 2 console.log("g'(2) = " + gPrime(2)) // 2*2 = 4

Deklarativ vs Imperativ programmering?

Deklarativ

Användaren är inloggad har mörkt tema på och är på startsidan.

Imperativ

Om användaren klickar på login tar vi oss till inloggade sidan. Om användaren klickar på mörkt tema hittar vi element och byter färg på dem.

Varför är deklarativ så bra?

  • Minskar risken för broblem som uppstår när vissa saker görs i en viss ordning.
  • Lättare att debugga eftersom man enkelt kan ladda in ett visst state.
  • Mindre rörig kod.
  • Lättare att implementera komponenter som är beroende på varandra.
